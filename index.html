脚本优化：

我需要大家帮忙完善题库，详情如下：
建议大家进入到答题页面，运行一下脚本，以防止重复；如果您有时间，可以顺便检查一下对错。

1.提交新题：
1.1.您的问题和答案需要确认过，并和Bilibili答题页面的源码没有任何偏差；
1.2.特殊字符：
1.2.1.html中的特殊字符根据网页的源码编写；
1.2.2.然后Java中的特殊字符用“\”注释；
1.3.将问题写成下面这种格式，一行一道，发给我：
case "问题": return "答案";
如：
问题一：一加一等于几？答案：二
写成上面的格式应该是：
case "一加一等于几？": return "二";
问题二："<a>"在html中是什么？答案：超链接
写成上面的格式应该是：
case "\"&lt;a&gt;\"在html中是什么？": return "超链接";
1.4.注意去掉问题和答案前后的空格。
1.5.您可以使用下面的题库生成器。

2.完善题库：
2.1.若您发现题库有错误、因为和Bilibili答题页面源码有偏差导致无法正确填答案或重复等问题，请将相关信息（问题正确名称、问题正确答案和代码行数位置等）发给我；
2.2.若问题的答案是“hyErrNotFound”，会导致核心模块判定问题在题库中没有找到；虽然应该不会出现，不过如果真的发生了，提醒我修改脚本的核心模块。

由于Google Chrome特别喜欢使用缓存而拒绝从服务器重新读取，我们建议您使用脚本时用“Developer Tools”禁用缓存；操作方法如下：
1.按“F12”打开“Developer Tools”；
2.点击“Developer Tools”右上角一个齿轮样的按钮；
3.在“General”下，勾选第一项（“Disable cache (while DevTools is open)”)。
完成以上操作后，您开着“Developer Tools”，缓存就会被禁用。
我们建议您使用答题脚本时开着“Developer Tools”，这样能每次都从云端重新读取最新版本的脚本。
